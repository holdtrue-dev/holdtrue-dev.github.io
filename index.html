the intent

plain language in

# intent: clamp

Clamp a number into a range. Given x, lo, hi, return x if it sits inside [lo, hi], otherwise return the nearest bound.

You write this. the rest is proof.

the contract

every line, in plain words

@deal.pre(lambda x, lo, hi: lo <= hi)
precondition. only promised for a valid range.
@deal.ensure(lambda x, lo, hi, result: lo <= result <= hi)
in range. the result lands inside [lo, hi].
@deal.ensure(lambda x, lo, hi, result: result == min(max(x, lo), hi))
exact value. it equals the clamped number.
@deal.raises()
no surprises. it raises nothing.
def clamp(x: int, lo: int, hi: int) -> int: ...
signature. the name and types to match.

the verdict

no answer without evidence

GUARANTEED

proven over all inputs, and the contract catches injected bugs and rejects broken stand-ins.

UNGUARANTEED

only sampled evidence. still needs human review.

FAILED

a counterexample, with the input that breaks it.

getting started

clone, sync, verify

# 1. clone and install (uv pins the whole toolchain)
git clone https://github.com/holdtrue-dev/holdtrue
cd holdtrue && uv sync

# 2. verify a correct implementation against its contract
uv run python -m holdtrue.cli verify examples/clamp \
              --impl examples/clamp/controls/correct.py

see it run

a real verification, start to finish

holdtrue verify

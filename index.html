prove it. don't read it.

holdtrue checks code against a contract you approve, instead of by reading the code. You review the contract; it tells you what is actually guaranteed.

#how

You write what you want in plain language. holdtrue turns it into a checkable contract: types, properties, and a spec a solver can prove. You approve it. An implementation is checked against the contract, and you get a verdict for each thing you asked for.

#verdicts

  • GUARANTEED: proven over every input, and the contract caught injected bugs. You do not need to read the code.
  • UNGUARANTEED: only sampled evidence. Still needs human review.
  • FAILED: a concrete input breaks the contract. You get that input.

$getting started

Python 3.12 and uv.

git clone https://github.com/holdtrue-dev/holdtrue
cd holdtrue
uv sync
uv run holdtrue verify examples/clamp \
  --impl examples/clamp/controls/correct.py

#scope

Today it covers Python pure functions over domains a solver can exhaust. Anything it cannot prove, like floats, loops, or IO, comes back UNGUARANTEED, said plainly. Whether the contract matches what you meant is still your call.
